Overview

This API provides a RESTful interface to GraphQL content types with dynamic routing and minimal validation. It transforms REST query parameters into GraphQL queries, making it easy to work with structured content without writing GraphQL.

Dynamic Routing

Accept any content type name - no hardcoded schemas required.

Powerful Filtering

Support for nested filters, logical operators, and complex queries.

Auto-Expand

Automatically expand scalar fields with intelligent depth control.

Type-Safe

Built with TypeScript and Zod validation for reliability.

Authentication

Published Routes: Use SingleKey authentication (public access)

Preview Routes: Require App Key + Secret authentication for draft/preview content

Configure authentication via environment variables (OPTIMIZELY_GRAPH_SINGLE_KEY, OPTIMIZELY_GRAPH_APP_KEY, etc.)

Published Content Routes

                    
Access published content using SingleKey authentication.


                    

                        

                            GET
                            /api/published/:contentType
                        

                        
Query published content by type with filtering, pagination, and sorting.

                        

                            
                                

                                    Parameter
                                    Type
                                    Description
                                

                            
                            
                                

                                    :contentType
                                    string
                                    Content type name (e.g., ArticlePage, Product, Hero)
                                

                            
                        

                    


                    

                        

                            GET
                            /api/published/:contentType/:id
                        

                        
Get a single published item by ID.

                        

                            
                                

                                    Parameter
                                    Type
                                    Description
                                

                            
                            
                                

                                    :contentType
                                    string
                                    Content type name
                                

                                

                                    :id
                                    string
                                    Content item ID
                                

                            
                        

                    


                    

                        

                            GET
                            /api/published/contentByPath
                        

                        
Get published content by URL path (routing).

                        

                            
                                

                                    Parameter
                                    Type
                                    Description
                                

                            
                            
                                

                                    url
                                    string
                                    URL path to match
                                

                                

                                    base
                                    string
                                    Base locale (optional)

Pagination

                    
Control result sets with pagination parameters:

                    
?limit=20&skip=10&cursor=abc123

                    

                        
                            

                                Parameter
                                Description
                            

                        
                        
                            

                                limit
                                Maximum number of items to return
                            

                            

                                skip
                                Number of items to skip (offset)
                            

                            

                                cursor
                                Cursor for cursor-based pagination
                            

                        
                    

                


                

                    
Filtering

                    
Use nested notation for complex filters:


                    
Simple Filters

                    
?where[Heading][eq]=Welcome
?where[Author][contains]=John
?where[_modified][gte]=2024-01-01


                    
Logical Operators

                    
?where[_and][0][field1][eq]=value1&where[_and][1][field2][eq]=value2
?where[_or][0][Status][eq]=Published&where[_or][1][Status][eq]=Draft


                    
Common Operators

                    

                        
                            

                                Operator
                                Description
                            

                        
                        
                            

                                eq
                                Equals
                            

                            

                                ne
                                Not equals
                            

                            

                                contains
                                Contains text
                            

                            

                                gte
                                Greater than or equal
                            

                            

                                lte
                                Less than or equal
                            

                            

                                exist
                                Field exists
                            

                        
                    

                


                

                    
Sorting

                    
Sort results by one or more fields:

                    
?orderBy[_modified]=DESC&orderBy[Heading]=ASC

                    
Use ASC for ascending or DESC for descending order.

                


                

                    
Advanced Options


                    
Localization

                    
?locale=en,fr


                    
Field Selection

                    
?fields=_id,Heading,Author


                    
Auto-Expand

                    
?expand=auto                    # Auto-expand scalar fields (excludes _fulltext)
?expand=auto_with_fulltext      # Auto-expand with _fulltext included
?expand=auto&depth=1            # Auto-expand with nested objects


                    
Multiple IDs

                    
?ids=id1,id2,id3


                    
Tracking

                    
?track=campaign_123


                    
Fragments (Polymorphic Types)

                    
?fragments=Hero,Card,Button

                


                

                    
Usage Examples


                    
List Articles

                    
GET /api/published/ArticlePage?limit=10&orderBy[_modified]=DESC


                    
Search with Filter

                    
GET /api/published/Hero?where[Heading][contains]=Welcome&locale=en


                    
Get Single Item

                    
GET /api/published/Product/abc123?fields=_id,Title,Price


                    
Get Content by URL Path

                    
GET /api/published/contentByPath?url=/products/laptop&base=en


                    
Query with Fragments

                    
GET /api/published/LandingPage?fragments=Hero,Card,Button


                    
Complex Query

                    
GET /api/published/LandingPage?where[_and][0][SeoSettings][exist]=true&where[_and][1][_modified][gte]=2024-01-01&limit=5


                    
Raw GraphQL

                    
POST /graphql
Content-Type: application/json

{
  "query": "query { ArticlePage { items { _id Heading } } }"
